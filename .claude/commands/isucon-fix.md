ISUCONアプリの指定された問題を修正してください。

引数: $ARGUMENTS

修正前に必ず以下を確認する:
0. `scripts/env.sh` を Read して環境情報を把握する（`ISUCON_WEBAPP_DIR`・`ISUCON_APP_LANG`・`DB_HOST`・`DB_USER`・`DB_PASS`・`DB_NAME`）
1. 該当ファイルをReadして現在の実装を把握する
2. 変更によってアプリの動作が壊れないか確認する（ベンチマークはfailスコアに厳しい）
3. 修正はシンプルに留める（余計な抽象化を入れない）

典型的な修正パターン:

**N+1 → JOIN/IN句**
- ループ内のSELECTを1つのJOINクエリに置き換える
- または先にIDリストを集めてIN句で一括取得する

**インデックス追加**
- ALTER TABLE / CREATE INDEX の SQL を生成する
- SQLディレクトリを確認してから置き場所を決める: `find . -name '*.sql' | head -10`
- 初期化スクリプト（`schema.sql` など）があればそこに追記するか、別ファイルで `add_indexes.sql` を作る

**画像をファイルシステムへ移動**
- まずアプリコードで静的ファイルの配信パスを確認する（`public/`、`static/`、`assets/` など環境ごとに異なる）
- DBからBLOBを読み出して確認したパスに書き出すスクリプトを作る
- アプリの画像取得ロジックをファイル読み込みに変更する
- nginxで直接配信するよう `templates/nginx-staticfile.conf` を参考にパスを合わせて適用する

**キャッシュ追加**
- まずアプリコードでキャッシュ層を確認する（Memcached / Redis のどちらを使っているか）
- 変更頻度が低いデータ（ユーザー情報など）をまとめて取得してキャッシュに乗せる
- キャッシュのTTLは短めに設定（競技ではデータ整合性チェックがある）

修正後:
- 変更内容を簡潔に説明する
- `git diff` で変更箇所を確認するよう促す
- ベンチマークを再実行して `scripts/score-log.sh <score> "<メモ>"` でスコアを記録するよう促す
