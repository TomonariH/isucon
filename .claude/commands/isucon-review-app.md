ISUCONアプリのコードを全ファイル読んで、パフォーマンス上の問題点をリストアップしてください。

手順:
1. まずWebアプリの実装ディレクトリを特定する（golang/ ruby/ python/ node/ php/ など）
2. メインアプリファイル（ルーティング・DBアクセス・ビジネスロジック）を全て読む
3. 以下のパターンを探して列挙する:

**N+1クエリ**
- ループ内でSELECTを発行しているコード
- 関連データを1件ずつ取得しているコード

**インデックス不足が疑われるクエリ**
- WHERE句に頻出するカラムへのインデックスがなさそうなSELECT
- ORDER BY で使っているカラムへのインデックス

**不要な処理・非効率なロジック**
- SELECT * で全カラム取得している（必要なカラムだけ取る）
- ループ内での重複DB呼び出し
- 結果をキャッシュできるが毎回DBから取っている処理

**その他の典型的な問題**
- 画像をDBのBLOBで管理している
- セッションをDBに保存している
- HTTP外部通信をループ内で行っている
